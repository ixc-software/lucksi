NAME

read - read from a file descriptor  

SYNOPSIS

#include <lwip/sockets.h>

ssize_t read(int fd, void *buf, size_t count);  

DESCRIPTION

read() attempts to read up to count bytes from file descriptor fd into the buffer starting at buf. If count is zero, read() returns zero and has no other results.

 

RETURN VALUE

On success, the number of bytes read is returned (zero indicates end of file). It is not an error if this number is smaller than the number of bytes requested; this may happen for example because fewer bytes are actually available right now (maybe because we were close to end-of-file, or because read() was interrupted. On error, -1 is returned, and specific error can be obtained by calling the function GetSocketError. In multi-threaded environment the last error number is held seperately for each thread.  

ERRORS

EINTR
The call was interrupted by a signal before any data was read. Not Supported
EAGAIN
Non-blocking I/O has been selected using O_NONBLOCK and no data was immediately available for reading. Not Supported
EIO
I/O error.
EINVAL
fd is attached to an object which is unsuitable for reading.
EFAULT
buf is outside your accessible address space. Not Supported
